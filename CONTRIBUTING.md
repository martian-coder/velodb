# Contributing to VeloDB

Thank you for your interest in contributing to VeloDB! We welcome contributions from the community to help make VeloDB the fastest and most reliable storage engine.

## How to Contribute

1.  **Report Bugs**: Open an issue on GitHub with a detailed description and steps to reproduce.
2.  **Suggest Features**: Share your ideas by opening a feature request issue.
3.  **Submit Pull Requests**:
    -   Fork the repository.
    -   Create a new branch for your changes.
    -   Follow the existing code style.
    -   Ensure all tests pass.
    -   Submit a PR with a clear description of your work.

## Contributor License Agreement (CLA)

By submitting a contribution to this project, you agree that your contribution is subject to the project's [LICENSE](LICENSE). 

Please note that for significant contributions, we may require you to sign a formal Contributor License Agreement (CLA) in the future to ensure the long-term protection and integrity of the VeloDB project.

## Code of Conduct

Help us keep VeloDB a welcoming environment for everyone. Please be respectful and professional in all interactions.
